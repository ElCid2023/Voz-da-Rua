Objetivo do Projeto

Sistema web para cidadãos registrarem reclamações sobre problemas urbanos em seus bairros, facilitando a comunicação com o poder público.

  • Sistema de autenticação (login/cadastro)
  • Cadastro de reclamações com localização por CEP
  • Visualização de reclamações em lista
  • Mapa interativo mostrando localização das reclamações
  • Painel administrativo

Problemas Resolvidos

1. Autenticação não Persistia

Problema: Ao clicar em "Nova Reclamação", pedia login mesmo estando logado.

Solução: Adicionar verificação de autenticação na função showSection().

2. CEP não Preenchia Campos

Problema: Campos com readonly não permitiam edição.

Solução: Remover atributo readonly dos campos de endereço.

3. Navegação não Funcionava

Problema: Links com href="#..." não acionavam JavaScript.

Solução: Trocar por onclick="showSection()".

4. Mapa não Aparecia

Problema: Faltava CSS para exibir seções.

Solução: Adicionar CSS .section { display: none; } e .section.active { display: block; }.

5. Erro CORS no Mapa

Problema: Nominatim bloqueava requisições do GitHub Pages.

Solução: Usar base de coordenadas conhecidas para CEPs de São Paulo.
